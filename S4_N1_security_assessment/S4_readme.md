# Supplementary Data: Representative security-assessment workflow

This folder provides the task prompt and generated workflow plan for the representative N-1 security-assessment workflow discussed in Section VI.

The purpose of this material is to show how a high-level engineering intent is converted into a scoped, engineer-supervised contingency-analysis workflow. The workflow first clarifies whether the N-1 assessment should be global or partial, then organizes the procedure into model/scenario activation, contingency-set configuration, sequential line-outage assessment, state restoration, report retrieval, and engineer review.

The files do not provide confidential grid topology, component names, electrical parameters, or raw contingency results. The reported workflow time is summarized in the main text; this supplementary material focuses on the task-level workflow structure.
